# Day时序 v1.0.0 结项交接

## 文档目的

记录 v1.0.0 的交付状态、已知限制和延期事项，供后续维护者参考。

## 已完成交付

### 功能模块

- 习惯管理（创建、编辑、归档、排序、内容安全检查）
- 每日打卡（打卡/取消、连续天数计算、日历视图、防抖）
- 冻结日（保护连续天数，月度限额 2 次）
- 灵感板（文本/清单笔记，内容安全检查）
- 仪式流（创建→关联习惯→计时执行→批量打卡）
- 旅程系统（预设旅程列表→步骤推进→完成庆祝）
- 时间轴（月历视图，历史打卡查看）
- 个人资料（微信头像/昵称同步）
- AI 洞察（周对比分析卡片）
- 统计详情（热力图、连续天数、周对比）
- 设置（减少动效、周起始、习惯提醒、数据导出、缓存清除、意见反馈）
- 首次启动引导流（4 场景）
- 订阅消息通知（活动进度提醒 + 每日打卡提醒）
- 版本自动更新提示（getUpdateManager）
- 分享卡片

### 工程基础设施

- 7 个云函数（ai、habit、journey、notify、ritual、stats、user）
- 一次性回填工具 backfill-streaks（频率感知连续天数回填，不在 `cloudbaserc.json` 常规部署配置中；详见 `cloudfunctions/backfill-streaks/README.md`）
- 云函数 CLI 部署流程（CloudBase CLI）
- 云函数 shared 模块同步机制（`cf:sync:shared` + 一致性校验）
- miniprogram-ci 自动上传
- 预飞检查脚本（10 项检查）
- 代码卫生检查脚本（debugger/localhost/console.log 等）
- 质量门禁（`npm run check:gate`）
- 带门禁的一键发布（`npm run release:guarded`）
- Jest 测试覆盖（habit、user、ritual、stats、notify、backfill-streaks、streak-parity）
- check_ins UTC+8 日期迁移指南（迁移云函数 migrate-utc8 已在迁移完成后从仓库移除，操作文档仍保留供参考）

## 未完成 / 延期事项

以下事项在当前仓库中未实现或未完整覆盖：

| 事项 | 说明 |
|------|------|
| Archive 独立云函数 | Archive 页面当前由前端聚合 check_ins 和 board_notes 生成，无独立后端 |
| 端到端测试 | 当前仅有云函数单元测试，无小程序端自动化测试 |
| CI/CD 流水线 | 发布依赖本地脚本执行，无远程 CI 集成 |
| 多环境切换 | 云环境 ID 硬编码在 cloudbaserc.json，无测试/生产环境隔离机制 |

## 已知限制与风险

| 限制 | 影响 | 缓解方式 |
|------|------|----------|
| Node 版本兼容性 | `package.json` 声明 `engines: 16.x`，但 `upload-miniprogram.js` 已内置 less@4→3 降级补丁以支持更高版本 | 升级 Node 时验证上传脚本补丁是否仍有效；如补丁失效，需手动处理 miniprogram-ci 的 less ESM 问题 |
| 云函数超时 | 批量操作在数据量大时可能超时 | 已按批次设计，注意监控 |
| 订阅消息模板 ID 硬编码 | 模板 ID 变更需同步修改 `cloudfunctions/notify/index.js` 和 `utils/constants.ts` | 变更时参照发布指南中的模板 ID 对照表 |
| 数据库索引需手动创建 | 无自动化索引管理 | 按发布指南中的索引列表手动创建 |

## 维护建议

1. **发布前始终运行 `npm run release:guarded`**，不要跳过质量门禁
2. **修改云函数后先运行 `npm run cf:sync:shared`**，确保 shared 模块一致
3. **关注 notify 云函数的定时触发器**，这是订阅消息的推送来源
4. **数据库备份保持开启**，发布指南中建议每日备份保留 7 天
5. **升级 Node 版本前**验证 `scripts/upload-miniprogram.js` 中的 less 兼容补丁是否仍然需要
