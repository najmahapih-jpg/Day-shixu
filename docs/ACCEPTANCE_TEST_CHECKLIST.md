# 发布前验收清单

## 使用方式

在准备提交审核或正式发布前，先跑工程门禁：

```bash
npm.cmd run env:list
npm.cmd run check:gate
npm.cmd run release:check
```

如果走了正式上传流程，还应确认：

- 结构化发布记录已生成到 `releases/history/<environment>/`
- 本次发布描述与版本号正确

## 核心流程

### 引导与登录

- [ ] 清理数据后首次启动，能正常进入引导流程
- [ ] 完成引导后能进入首页
- [ ] 首次登录能正常创建用户数据

### 习惯管理

- [ ] 新建习惯成功

- [ ] 归档与恢复成功
- [ ] 排序调整后刷新仍保持

### 打卡与冻结

- [ ] 打卡成功，连续天数正确变化

- [ ] 冻结逻辑按预期工作

- [ ] timeline 页面正常打开

### 灵感板
